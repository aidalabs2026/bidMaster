\# \[PRD] ETRI 소액 입찰 금액 산정 및 시뮬레이션 대시보드 개발



\## 1. 프로젝트 개요

본 프로젝트는 ETRI(한국전자통신연구원)의 소액 입찰 낙찰자 결정 방식을 시뮬레이션하고, 데이터 기반의 최적 투찰 금액을 산정하는 UI 기반 프로그램을 개발하는 것을 목적으로 함.



\## 2. 핵심 비즈니스 로직 (ETRI 규정 준수)



\### 2.1 예비가격 생성 로직

\- \*\*기초금액($B$):\*\* 사용자가 입력한 공고상의 금액.

\- \*\*복수예비가격 생성:\*\* 전자입찰시스템의 난수발생기를 모사하여 기초금액의 $\\pm 2\\%$ 범위 내에서 총 15개의 예비가격을 생성함.

&#x20;   - 범위: $B \\times 0.98 \\le \\text{예비가격} \\le B \\times 1.02$



\### 2.2 예정가격 산출 로직

\- \*\*번호 추첨:\*\* 입찰자가 15개 번호 중 2개를 선택.

\- \*\*최종 4개 선정:\*\* 전체 입찰자가 선택한 번호 중 다수 추첨된 번호순으로 4개를 선정. (프로그램에서는 무작위 4개 추출로 시뮬레이션)

\- \*\*산술 평균:\*\* 선정된 4개 예비가격의 합계를 4로 나누어 예정가격($P$) 확정.

&#x20;   - $P = \\frac{\\sum\_{i=1}^{4} \\text{Selected Price}\_i}{4}$



\### 2.3 낙찰자 결정 로직 (소액계약)

\- \*\*낙찰하한율:\*\* 예정가격의 \*\*88%\*\* 적용.

\- \*\*낙찰 조건:\*\* 투찰 금액 $\\ge (P \\times 0.88)$ 조건을 만족하는 최저가 투찰자.



\---



\## 3. 상세 기능 요구사항 (Functional Requirements)



\### F-101: 사용자 입력 제어

\- \[ ] 기초금액(KRW) 입력 필드.

\- \[ ] 시뮬레이션 실행 횟수 설정 (예: 1,000 \~ 10,000회).

\- \[ ] 낙찰하한율 사용자 설정 (기본값 88%).



\### F-102: 시뮬레이션 엔진

\- \[ ] 15개 복수예비가격 무작위 생성 알고리즘.

\- \[ ] 반복 시뮬레이션을 통한 예정가격 분포 데이터 생성.

\- \[ ] 산출된 예정가격 중 빈도수가 높은 구간(사정률) 분석.



\### F-103: UI/UX 시각화 (중요)

\- \[ ] \*\*히스토그램:\*\* 시뮬레이션 결과(예정가격 분포)를 막대그래프로 시각화.

\- \[ ] \*\*지표 카드:\*\* 최저/최고 예정가격, 평균 예정가격, 표준편차 표시.

\- \[ ] \*\*결과 강조:\*\* 최종 추천 투찰 금액(평균 예정가격 × 88%)을 크게 강조.

\## 4. 기술 스택 제안

\- \*\*Language:\*\* Python

\- \*\*UI Framework:\*\* Streamlit (웹 기반 대시보드) 또는 PyQt5 (데스크톱 앱).

\- \*\*Visualization:\*\* Matplotlib, Plotly 또는 Seaborn.



\## 5. 최종 출력물 가이드 (Claude 요청사항)

1\. 위 로직을 모두 포함한 \*\*Python 실행 코드\*\*를 작성할 것.

2\. 사용자가 브라우저에서 인터랙티브하게 조작할 수 있도록 \*\*Streamlit\*\* 라이브러리를 우선적으로 사용할 것.

3\. 15개의 복수예비가격 리스트를 테이블 형태로 화면에 보여줄 것.

4\. 시뮬레이션 반복 횟수에 따른 예정가격의 수렴 과정을 시각적으로 잘 나타낼 것.
